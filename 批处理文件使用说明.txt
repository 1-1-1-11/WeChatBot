# 批处理文件使用说明

## ✅ 已修复的问题

1. **乱码问题** - 所有脚本已使用英文文本，避免编码问题
2. **闪退问题** - 添加了错误处理和暂停，出错时不会立即关闭
3. **稳定性** - 每个命令都有错误检查和友好提示

## 🚀 推荐使用方式

### 方案 1：使用英文版脚本（推荐）

```bash
# 首次配置
config-wizard.bat

# 日常测试
快速启动.bat

# 正式使用
生产模式启动.bat

# 完整检查
start.bat
```

### 方案 2：直接用命令行（最稳定）

打开 PowerShell 或 CMD，在项目目录执行：

```powershell
# 首次配置：复制并编辑配置文件
copy .env.example .env
notepad .env
# 修改以下内容：
#   OPENAI_BASE_URL=https://api.openai.com
#   OPENAI_API_KEY=sk-your-key-here
#   MODEL_NAME=gpt-3.5-turbo
#   DRY_RUN=true

# 安装依赖
python -m pip install -r requirements.txt

# 测试模式启动
python -m wechat_bot --env .env

# 生产模式启动（修改 .env 中 DRY_RUN=false）
python -m wechat_bot --env .env
```

## 📝 配置文件编辑（.env）

用记事本打开 `.env` 文件，修改以下内容：

```env
# API 配置
OPENAI_BASE_URL=https://api.openai.com
OPENAI_API_KEY=sk-your-actual-api-key-here
MODEL_NAME=gpt-3.5-turbo

# 行为配置
IDLE_MINUTES=10                      # 空闲多少分钟触发离线模式
AUTO_REPLY_DELAY_MIN_SECONDS=20      # 自动回复最小延迟
AUTO_REPLY_DELAY_MAX_SECONDS=60      # 自动回复最大延迟
CONTACT_COOLDOWN_MINUTES=10          # 同一联系人冷却时间

# 数据目录
DATA_DIR=data

# 运行模式（重要！）
DRY_RUN=true                         # true=测试模式，false=生产模式
```

**重要提示**：
- 首次使用保持 `DRY_RUN=true`
- 充分测试后再改为 `DRY_RUN=false`

## 🧪 测试流程

### 1. 快速测试（推荐）

```powershell
# 确保 DRY_RUN=true
python -m wechat_bot.app --env .env --smoke-test
```

看到 `smoke ok` 表示配置正确。

### 2. 现场测试

```powershell
# 测试模式启动
python -m wechat_bot --env .env
```

然后：
1. 让电脑空闲 10 分钟（不动鼠标键盘）
2. 让测试联系人发消息："在吗"
3. 20-60 秒后查看控制台"自动回复"标签页
4. 应该看到记录但不会真实发送

## 🚀 正式使用

确认测试成功后：

1. 编辑 `.env`，设置 `DRY_RUN=false`
2. 运行：
   ```powershell
   python -m wechat_bot --env .env
   ```
3. 会真实发送微信消息

## ⚠️ 常见问题

### Q1: 批处理文件还是闪退？

**解决方法**：
- 右键脚本 → "以管理员身份运行"
- 或使用 PowerShell：
  ```powershell
  cd D:\Desktop\WechatBot
  .\start.bat
  ```

### Q2: 提示 "Python not found"？

**解决方法**：
1. 下载安装 Python 3.9+：https://www.python.org/downloads/
2. 安装时勾选 "Add Python to PATH"
3. 重启命令行窗口

### Q3: 提示依赖包缺失？

**解决方法**：
```powershell
python -m pip install -r requirements.txt
```

### Q4: 如何检查是否真的在运行？

**方法**：
1. 启动后会看到控制台窗口（不会关闭）
2. 顶部显示状态："在线汇总" 或 "离线值班"
3. 标签页可以切换查看日志
