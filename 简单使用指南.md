# WeChatBot 简单使用指南

## 批处理文件有乱码问题 - 推荐使用英文版

由于 Windows 批处理文件的编码问题，中文版本可能显示乱码。
**推荐使用英文版本的批处理文件，它们功能完全相同且更稳定。**

---

## 🚀 快速开始（3步）

### 方法 1：使用批处理文件（推荐）

#### 第 1 步：配置
双击运行：**`setup-wizard.bat`** （配置向导 - 英文界面）

按提示输入：
- API URL: 你的 API 地址（如 `https://api.openai.com`）
- API Key: 你的 API 密钥（如 `sk-xxx`）
- Model: 选择 1（gpt-3.5-turbo，推荐）
- 其他选项：直接回车使用默认值
- Mode: 选择 1（Test 测试模式）

#### 第 2 步：配置微信（只需一次）
1. 按 `Win + Ctrl + Enter` 启动 Windows 讲述人
2. 登录微信（如果未登录）
3. 等待 5 分钟
4. 按 `Win + Ctrl + Enter` 关闭讲述人

#### 第 3 步：启动
- **测试模式**：双击 `quick-start.bat`（不会真实发送消息）
- **正式使用**：双击 `production-start.bat`（会真实发送消息）

---

### 方法 2：直接用命令行（最简单）

打开 PowerShell，在项目目录运行：

```powershell
# 1. 创建并编辑配置文件
copy .env.example .env
notepad .env
```

编辑 `.env` 文件，修改以下内容：
```
OPENAI_BASE_URL=https://api.openai.com
OPENAI_API_KEY=sk-你的密钥
MODEL_NAME=gpt-3.5-turbo
DRY_RUN=true
```

```powershell
# 2. 安装依赖
python -m pip install -r requirements.txt

# 3. 启动（测试模式）
python -m wechat_bot --env .env
```

## 📁 批处理文件说明

| 英文文件名 | 中文说明 | 功能 |
|-----------|---------|------|
| `setup-wizard.bat` | 配置向导 | 交互式配置，首次使用 |
| `quick-start.bat` | 快速启动 | 测试模式，不发送消息 |
| `production-start.bat` | 生产启动 | 真实发送消息 |
| `full-start.bat` | 完整启动 | 包含所有检查 |

**注意**：旧的中文文件名批处理（配置向导.bat、快速启动.bat 等）有编码问题，请使用上面的英文版本。

## ⚙️ 配置说明（.env 文件）

用记事本打开 `.env` 文件，需要修改的内容：

```env
# ===== 必须修改 =====
OPENAI_BASE_URL=https://api.openai.com    # 你的 API 地址
OPENAI_API_KEY=sk-your-key-here           # 你的 API 密钥
MODEL_NAME=gpt-3.5-turbo                  # 模型名称

# ===== 可选修改 =====
IDLE_MINUTES=10                           # 空闲多少分钟触发离线
AUTO_REPLY_DELAY_MIN_SECONDS=20           # 回复最小延迟
AUTO_REPLY_DELAY_MAX_SECONDS=60           # 回复最大延迟
CONTACT_COOLDOWN_MINUTES=10               # 同一人冷却时间

# ===== 重要！=====
DRY_RUN=true                              # true=测试，false=真实发送
```

## 🎮 控制台界面

启动后会看到控制台窗口：

```
┌──────────────────────────────────────────────┐
│ 状态：离线值班 | 模式：auto | 空闲：650秒    │
├──────────────────────────────────────────────┤
│ [ ] 暂停自动回复                             │
│ (•) 自动检测  ( ) 强制在线  ( ) 强制离线     │
├──────────────────────────────────────────────┤
│ [每日总览] [待办风险] [自动回复]              │
└──────────────────────────────────────────────┘
```

**说明**：
- **暂停自动回复**：勾选后停止自动回复
- **自动检测**：根据鼠标键盘自动判断（推荐）
- **强制在线**：不自动回复
- **强制离线**：允许自动回复

---

## ✅ 自动回复触发条件

必须**同时满足**所有条件：

1. ✅ 电脑空闲 ≥ 10 分钟（默认）
2. ✅ 个人消息（不是群聊）
3. ✅ 低风险消息（不含"报价"、"转账"等）
4. ✅ 距上次回复 ≥ 10 分钟
5. ✅ 未勾选"暂停自动回复"
6. ✅ DRY_RUN=false（生产模式）

## 🧪 测试建议

### 测试模式验证
1. 确保 `.env` 中 `DRY_RUN=true`
2. 运行 `quick-start.bat`
3. 让电脑空闲 10 分钟
4. 测试联系人发消息："在吗"
5. 查看控制台"自动回复"标签页
6. 应该看到记录但不会真实发送

### 真实发送验证
1. 确认测试成功
2. 运行 `production-start.bat`
3. 会自动设置 `DRY_RUN=false`
4. 重复上述测试步骤
5. 测试联系人会收到自动回复

## ❓ 常见问题

### Q1: 批处理文件显示乱码？
**答**：使用英文版本：`setup-wizard.bat`、`quick-start.bat`、`production-start.bat`

### Q2: 提示"Python not found"？
**答**：
1. 下载 Python 3.9+：https://www.python.org/downloads/
2. 安装时勾选"Add Python to PATH"

### Q3: 为什么不自动回复？
**答**：检查：
- 空闲时间够了吗？
- 是个人消息吗？
- DRY_RUN=false 吗？
- 勾选"暂停自动回复"了吗？

### Q4: 如何停止？
**答**：关闭控制台窗口，或按 `Ctrl+C`
